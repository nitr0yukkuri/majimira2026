# プロジェクト仕様書：Neo-City Awakening (Magical Mirai 2026)

## 1. コンセプト定義
### 「楽曲が、都市の鼓動を再起動させる」
歌詞を単なる文字情報としてではなく、都市を構成する「データ・ストリーム」として定義。最初はグリッチやノイズに塗れた暗いワイヤーフレームの世界が、歌声と共にネオン輝くサイバーパンク都市へと変貌を遂げる。

---

## 2. 要件定義

### 2.1 機能要件
1. **リアルタイム同期システム**
   - TextAlive App APIを用いた、歌詞、ビート、サビ（Chorus）フラグとの完全同期。
2. **動的ライティング演出**
   - 歌い出し（Aメロ）で特定のビルや信号機が点灯。
   - サビ（Chorus）で街全体のBloomエフェクト（発光）を最大化。
3. **都市の再構築演出（GLSL）**
   - 暗黒のワイヤーフレームから、マテリアルが実体化するトランジション演出。
4. **カメラワーク・オートメーション**
   - 曲のセクション（Intro/Verse/Chorus）に応じたカメラパスの切り替え。
   - サビでの俯瞰視点（ドローンショット）への移行。

### 2.2 非機能要件
1. **パフォーマンス（60FPS維持）**
   - 大量のオブジェクトを扱うため、InstancedMeshやポストプロセスの最適化。
2. **ブラウザ互換性**
   - WebGL2.0をサポートする主要ブラウザでの動作。

## 4. 技術実装詳細

### 4.1 歌詞とオブジェクトの紐付け
TextAliveの `IPhrase` や `IWord` イベントをフックし、空間内の `Building` コンポーネントの `emissive` 強度を操作。
- `char.startTime` に合わせて、対応するビルの窓や看板のテクスチャを1文字ずつ光らせる。

### 4.2 GLSLによる「Before / After」の表現
単純な透明度の変更ではなく、シェーダーで以下のステップを踏む：
1. **Noise Phase:** `Simplex Noise` を用いて、ビルがデジタルノイズに溶けている状態を描画。
2. **Reconstruction:** 音圧（Loudness）に応じて、ノイズの閾値を下げ、ソリッドなポリゴンを表示。
3. **Emission:** 最終的に `UnrealBloomPass` でネオンの輝きを付与。

### 4.3 パフォーマンス最適化戦略
- **Instanced Mesh:** 1000個以上の窓やビルを、単一の描画コールで処理。
- **Post-processing:** Bloomは重いため、解像度を抑えた `Luminance Mask` を使用。

## 5. シーン構成案（タイムライン）

| セクション | 視覚演出 | カメラワーク |
| :--- | :--- | :--- |
| **Intro** | 真っ暗な街、ワイヤーフレーム。 | 地面スレスレの低速移動。 |
| **Aメロ** | 歌詞に合わせて特定の窓が淡く点滅。 | 街路を抜ける一人称視点。 |
| **Bメロ** | ノイズが減り、街の色（シアン/マゼンタ）が徐々に現れる。 | 街を見上げるローアングル。 |
| **Chorus** | 都市全体がフルブライト。全発光。 | 上空へ急上昇し、都市全体を俯瞰。 |
| **Outro** | 光が収束し、一つの回路図となって消えていく。 | ズームアウトし、ロゴを表示。 |
